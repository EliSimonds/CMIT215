Process Explorer, what is it?

Process Explorer is a Freeware replacement to the Task Management tool for Microsoft windows. This tool was created by Sysinternals a Microsoft Subsidiary in 2001. And is now on version 16.1, which was updated on January 4 2016.


System requirements?

Windows XP, Vista and later or Windows server 2008 or later.


The installation processes.

The Process Explorer files can easily be obtained for free from many online sources. For the sake of this project I will be using the fallowing source: https://technet.microsoft.com/en-us/sysinternals/processexplorer

Upon opening procexp.exe, the main Process Explorer window will open. The entire program runs from the 2.5mb executable file, so no installation is required and the program can be put in to immediate use.



What's it offer above the standard Task Manager?

+ Running Process Explorer offers many benefits, beyond what the standard task manager can provide. A first look at Process Explorer main window will make many improved functions apparent.

+ The color coding differentiates between user and service processes and can be used as notifications of new and deleted processes. These colors can be altered under the options > Configure Colors.



+ Detailed view and sorting of running processes in the main display can be altered by adding data columns depending on the needed functions. Process Explorer offers a wide variety of options for this including CPU usage by thread, memory usage, or file location. This is accessed by right clicking the columns titles.



+ By default processes are sorted hierarchical. This allows a view of process trees and interaction with sub processes.

+ Mousing over a process displays a view of the command line, path location and ddl for quick browsing.

+ Detailed process information and properties can be accessed by double clicking any process row. Available information on individual processes and it's resource usage includes from standard file properties, CPU, virtual/physical Memory, GPU, TCP/IP, and local group policy's.



+ A variety of process priority options are available by right clicking a process row. Priority settings range from real time to background and idle. As well as the options to restart or suspend an active process.

+ Process Explorer allows users to directly search processes .exe files online for more information about their purpose, or to directly send them to VirusTotal.com to be live scanned.

+ DLL/Handle query options are available. Allowing users to see what the process is interacting with on the system and display their locations. A secondary window under the processes window can be toggled under options and customized to view DLL's/Handle's attributed to a selected process row.



+ Process Explorer can replace Task Manager as default for the ctrl + alt + del/ctrl + shift + esc commands. Not recommended as it will not be running as administer and may limit usage when interacting with higher level processes.


When would you use it?

+ If you needed to Handle leaks of system resources by locating DLL/Handles.

+ If you wanted to manage the usage of processes across multiple users and implement group policy's.

+ As your default Task Manager
